# Execution Strategy Patterns

## MVP First
1. Finish Setup
2. Finish Foundational
3. Finish User Story 1
4. Stop and validate MVP independently
5. Deploy or demo before continuing

## Incremental Delivery
1. Ship US1
2. Gather feedback
3. Ship US2
4. Gather feedback
5. Ship US3 or later-phase work only if still valuable

## Parallel Team Strategy
If multiple developers are available:
- finish Setup + Foundational together first
- then split by independent user story or clearly parallel-safe tasks
- rejoin for Polish, validation, and release prep

## Escalation rule
If a hidden dependency or >4 hour task appears, split or replan instead of letting the spec-kit task list drift silently.
